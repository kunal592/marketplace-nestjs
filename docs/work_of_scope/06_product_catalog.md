# Product Catalog

## Categories

Categories support nested hierarchy.

Example:

Electronics
├ Phones
└ Laptops

---

# Products

Products belong to:

vendor
category

Product fields:

name
slug
description
price
discount price
stock
images

---

# Product Variants

Variants support attributes like:

size
color
storage

Example:

T-Shirt

size: S M L
color: red blue

Variants must store stock and price.
